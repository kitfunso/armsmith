# Phase 2: The Agent Loop, Report, Repro, Provisioner

**Goal:** Turn the Spike-0 GO into a working end-to-end tool — the deterministic tuner + LLM analyst discovery loop, a self-contained HTML report, deterministic `repro` replay, and a turnkey provisioner — producing a CI-significant, quality-guarded, reproducible speedup on Graviton4 `r8g`.
**Prerequisite:** Phase 1 done (Spike 0 GO recorded; honest baseline captured; action registry stubbed). Spike verdict is **GO with loop framing = "try-measure-keep-best"** (`code_hotspots` only; counters `None` on virtualized `r8g`).
**The contract:** every module below is pinned in `docs/CONTRACTS.md`. Build to those exact types/signatures; do not invent shapes.
**Scope:** 11 steps. Steps 1-4 are offline (fixture-testable on the x86 PC); steps 5-11 need a live Graviton target. **Rule 1 stands: never report a number measured off a real Arm target.**

> Two tracks, do not let one block the other: the **offline core** (models → bench → actions → profiler-parse → report, all unit-tested against `tests/fixtures/`) can be built and green on the PC with zero AWS spend. The **on-target** work (target.py, real bench, discovery run, repro) gates on a live `r8g`. Build the offline core first so the Graviton sessions are short and cheap (rule 8: tear down after each).

## Step 7: `agent.py` — the discovery loop, keep/revert, trajectory, recipe, replay
**Files:** `src/armsmith/agent.py`, `tests/test_agent.py`
**What:** `enumerate_candidates`, `optimize` (greedy COORDINATE ASCENT — candidates recompose onto the current incumbent so kept levers STACK), `quality_ok`, `gap`, `reorder_by_priority`, `hoist_suggested`, `replay` (CONTRACTS §9). Loop: snapshot → brain reorders/hoists queue → pop → screen → (gate) `confirm_ab` (A/B/A/B vs incumbent, BOTH fresh) → keep iff CI-significant decode gain AND quality ok → regenerate queue off the new incumbent → `TrajectoryStep` to JSONL → `Recipe` at end. `replay` = no brain/profiler, rebuild+confirm, assert `tol_pct`.
**Verify:** `pytest tests/test_agent.py` with **fake `Target`/`Profiler`/`Benchmarker`** (fixture-driven, offline):
- a candidate that improves decode past the screen gate and confirms CI-significant (via `confirm_ab`, both results fresh) with good quality is **kept**; the incumbent advances; a `TrajectoryStep(kept=True)` is appended.
- two levers that each independently confirm CI-significant with good quality both **stack** onto the incumbent (coordinate ascent recomposes the queue after each keep) — the winning recipe carries multiple levers, not one.
- a candidate that improves speed but regresses quality past threshold is **reverted** (`kept=False`, `quality_ok=False`) — the quality guard bites (a quant-changing build lever with unmeasured quality is also rejected via `changed_quant`).
- a candidate that fails the screen gate never triggers a `confirm_ab` (cost control).
- with `NullBrain` + `NullProfiler` the loop converges to a winner **within CI of** the brain-guided run (the honesty test — "no worse, within CI", not bit-identical; brain only reorders/hoists) — CONTRACTS §9 budget-vs-grid.
- `optimize` writes `trajectory.jsonl` (one line per step) and `recipe.json`; `recipe.gap_closed_pct` computed vs `expert`.
- `replay(recipe)` within `tol_pct` passes; a fake result outside tol raises `ReproToleranceError`.
**Commit:** `feat: agent discovery loop + keep/revert + trajectory + recipe + replay`
